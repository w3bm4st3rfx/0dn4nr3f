What is the downside of using Expo?

Expo has some serious drawbacks that can limit the app growth. One of these limitations is dealing with native code as a developer - unlike the React Native CLI.

When you develop your app using Expo, you don't change the native code and this will be very frustrating news for developers who want to develop their own native modules and integrate them into their React Native apps.

One solution is to eject your app made in Expo to the React Native CLI, so you can mess with the Android and iOS native code directly. But once that ejection is done, it cannot go back and be an Expo project. This ejection can cause some incompatibilities and bugs in the App as well.

Using Expo depending on the project can limit development. For example, if your App needs to use bluetooth, then Expo will not be able to help you with that, as it is not yet supported.

Despite the great features it offers its users, Expo has some serious limitations that you need to consider when developing your next app.
